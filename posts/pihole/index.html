Pihole

21 Jan 2022 -

Green (easiest): install Pi-hole on a raspberry pi running raspbian

Installation of Raspbian/Raspberry Pi OS:

If you have access to a monitor or TV, you can use the SD card in your CanaKit, since that comes pre-loaded and is easy to install per these instructions in the “booting from NOOBS” section.

If you don’t have access to a monitor, or don’t want to use one, you can download a Raspberry Pi OS installer here. Once the install process is done, go into the /boot folder on the card and create a blank file named ssh. This will allow you to ssh into your pi by typing

1
2

$ ssh pi@<ip of raspberry pi>
Password: raspberry

(If you get asked any other questions, type yes)

Installation of Pi-hole:

Null-Byte has an amazing guide on it that should be easy to follow.

Please note that different routers will have different places to set the preferred DNS server. Mine was in the DHCP options, not the DNS or WAN options where you would expect it. In addition, it was hidden until I clicked custom servers.

If you are using a router that doesn’t allow you to set a DNS server from your local network, you can try setting it as your DHCP Server, as in Step 9 of Null-Byte’s guide

Blue (intermediate): install Pi-hole using docker on raspbian

Note: you cannot use this method if you can’t use the pi-hole as a DNS server. Setting up DHCP through a docker container is very difficult. If this doesn’t dissuade you, there’s a link to some info here

If doing this after the Green level, open a terminal on your pi and type pihole uninstall. This will make sure there’s only one instance of pihole running. Feel free to keep the dependencies installed by typing N at the prompt. Most are used for many projects.

Install docker:

Unfortunately, Debian’s repositories don’t include docker, so we’ll have to use an install script from docker.

To do so, run these two commands in a terminal:

1
2

$ curl -fsSL https://get.docker.com -o get-docker.sh
$ sudo sh get-docker.sh

Install docker-compose:

Docker compose lets you easily spin up docker containers by using .yml files which contain all the needed variables. Otherwise, you’d have to copy and paste a giant command or use a confusing bash script which doesn’t help.
For the same reasons, we’ll have to install docker-compose like this:

1
2
3

$ sudo apt-get update
$ sudo apt-get install python3-venv python3-pip
$ sudo pip3 install docker-compose

Here’s the pihole docker-compose.yml file (execute this command on your pi):

curl -fsSL https://lancol.tech/assets/files/docker-compose.yml -o docker-compose.yml

Make sure to change the WEBPASSWORD field to something more secure.

Use the command
sudo docker-compose up -d
While you have the docker-compose.yml file in your current directory

Black (Advanced): Install pi-hole using docker on Arch

For this one you’ll have to use your raspberry pi, another linux machine, Windows Subsystem for Linux, or some sort of BASH shell. If using the raspberry pi, you’ll have to use a separate SD card and flash it through the pi’s operating system.
For this I recommend using the Arch-Arm official guide. They leave out that you should probably just switch to your root user because all of the commands are going to use root privileges, and for some it’s a requirement to be the actual root user. To do that type:
sudo su -
Then follow the guide from the Arch-Arm website(ARM v7 installation section):

Be careful when referring to the root directory that you make when mounting your SD card. I used a ./root to refer to the one inside the current directory, making sure I never used /root, as there is another directory for the root user in that location, and you’ll want to keep that as it is. You could also just name ./root and ./boot ./1 and ./2 if you can keep track, to avoid that issue.
You can get out of root mode on your pi by typing:
exit
Then you should return to the $ prompt
In the windows terminal there should be a standard ssh applet that you can use to connect to the pi after installing the OS.

1

$ ssh alarm@<ip of raspberry pi>    #Password: alarm

After all of that installation troubles that didn’t exist on raspbian, there are more installation troubles that didn’t exist on raspbian! (pihole is pretty much exclusively tested on it, and docker is an edge case inside of an edge case)
The commands for that is as follows:

1
2
3
4
5
6
7

$ su -                                #then root password
# pacman-key --init
# pacman-key --populate archlinuxarm  #If you didn’t do that during the installation
# pacman -Syu --noconfirm             #To update the system in one command
# Pacman -S --needed docker docker-compose base-devel
# systemctl enable docker
# reboot

To install There should be a few errors. This is because there’s some parts of the docker-compose file that aren’t supported in this distribution.

1
2

# mkdir etc-dnsmasq.d
# mkdir etc-pihole

…Now there’s another error that looks more terrifying
Here’s how we fix it

1

# ss -lp 'sport = :domain'

See how this process called systemd-resolve is hogging those ports? That’s the default dns for this distribution of linux.

1
2

# systemctl stop systemd-resolved  
# systemctl disable systemd-resolved

This frees up port 53 for our dns shenanigans.
Now let’s try to start up the container again.

1

# docker-compose up -d

We replace systemd-resolve’s port with dnsmasq, but we won’t be able to use the port without telling dnsmasq where to bind the dns. Here’s how we do that:

1

# nano /etc/dnsmasq.conf

Then type
bind-interfaces 127.0.0.1:53
Then press ctrl-x then Y then ENTER to save the file.

1
2

# docker-compose up -d  
# reboot

This should fix the dns issues
After that, all that’s left is to change your DNS settings on your router to use the pi-hole as your local DNS.
Then you’re done! Navigate to /admin to manage your pi-hole! Might be a good idea to do

1
2

$ su -                       #then root password
# visudo

Then uncomment the line with %wheel in it, esc :wq!

1
2

# passwd -l root
# passwd alarm

As well as some other hardening of the system.
https://kyau.net/wiki/ArchLinux:Security seems to have a pretty good guide, although they’re using an alternate version of Pacman that can throw people off.
There’s also more information on the arch wiki:
https://wiki.archlinux.org/title/Security
Although the arch wiki is a little hard to read at times.
However if you just do the steps above then you’ll be in pretty decent shape, barring any vulnerabilities in ssh or the pihole instance that you don’t patch.
